Preguntas a responder

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?

    • A pesar de que son conceptos que se pueden llegar a confundir, no son lo mismo. Internet se refiere a la infraestructura global que conecta diferentes dispositivos entre diferentes redes (la red de redes) y usa tecnologías como TCP/IP o DNS. La web (WWW) se refiere a una parte del Internet que es accesible por navegadores web y consiste de páginas web, con el texto e imágenes que estas tengan.

  • ¿Cuáles son las partes de una URL?

    • Una URL está formada por el esquema o protocolo (normalmente HTTPS o HTTP, aunque existen otros), el dominio (como podría ser google.com, por ejemplo), opcionalmente un puerto (aunque HTTPS y HTTP ya lo traen incluido), una ruta (para indicar el lugar exacto dentro de una página web) y, opcionalmente, parámentros de consulta (indicados con "?") y anclas (indicadas con "#").

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • GET: Obtener recursos sin realizar modificación alguna.
    • HEAD: Muy similar al GET, pero solo para obtener información del encabezado de los recursos.
    • POST: Se usa para enviar datos al servidor con el objetivo de crear o actualizar recursos.
    • PUT: Reemplaza completamente un recurso existente. Si no hay uno, crea uno desde cero.
    • PATCH: Muy similar a PUT, pero solo modifica los recursos parcialmente. Muy útil para modificar tan sólo pequeños atributos de un recursos sin sobrescribirlo por completo.
    • DELETE: Elimina recursos del servidor.

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    • El método de POST. Porque se debe enviar datos al servidor para verificar si los datos de acceso proporcionados son correctos o, además, no se desea realizar modificación alguna de ningún recurso.

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    • El GET. Ya que el recurso se carga (a través del navegador web), pero este no debe sufrir modificación alguna.

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • Significa que la solicitud ha sido exitosa; entonces, no ha ocurrido ningún error.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • Excepto si el proveedor de hosting tiene algún problema, SÍ, es responsabilidad del desarrollador, ya que este error implica que, por algún motivo, la página no ha sido encontrada. Para esto, se debería redireccionar permanentemente o (si, por lo que sea no se puede) crear un página de 404 personalizada y que rediriga fácilmente a los usuarios a las otras secciones (que sí funcionan) de la página en cuestión.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • Cómo en el error 404, generalmete SÍ es responsabilidad del desarrollador. Este error significa que el servidor ha tenido un error interno, esto puede ser provocado por un error de código, el servidor web, la base de datos o tiempos de inactividad (para servicios externos), entre otros. El desarrollador debe de asegurarse de que su servidor funciona correctamente para dar a los clientes una buena experiencia.

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Es un elemento que, aunque podría ser posible seguirlo usando, ya no se recomienda utlizarlo en la actualidad, debido a que pronto podría quedar obsoleto; se recomienda usar equivalentes modernos. Por ejemplo, en lugar de <font> para cambiar tipos de fuentes, ahora se recomienda hacer esto desde CSS. En lugar de usar <big> y <small> se recomienda usar font-size desde CSS.

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • La nueva versión (HTML 5) contiene semántica mejorada, soporte multimedia nativo, soporte gráfico (como WebGL) nativo, más tipos de entradas para formularios, almacenamiento local, comptatibilidad móvil y más APIs nativas.

  • ¿Qué componentes de estructura y estilo tiene una tabla?

    • Una tabla tiene varios elementos que la componen. El contenedor principal se define con <table>. Para el título se usa <caption>. El encabezado, cuerpo y pie se definen con las etiquetas normales en HTML para esto, pero con una "t" por delante (<thead>, por ejemplo). Las filas son <tr>, una celda de encabezado con <th> y una celda de datos <td>. Se usan propiedades de CSS para configurar el estilo de las tablas, como border, además de propiedades "típicas" como color, background-color, font, font-size, por ejemplo.

  • ¿Cuáles son los principales controles de una forma HTML5?

    • El control más importante, por su versatilidad, de un formulario es <input>. Este se adapta a muchos tipos de datos para introducir, mediante type. Otros importantes son <select> (para crear menús desplegables), <option> (para dar opciones de dichos menús), <button> (para botones) y <label> (para etiquetas), entre otros.

  • ¿Cuál es el ciclo de vida de los sistemas de información?

    • Un sistema de información tiene diversas fases a lo largo de su vida:

    1. Planificación y estudio de viabilidad: Se examina cómo se necesita el sistema y si es viable crearlo.
    2. Análisis de requerimientos: Se recopilan necesidades, especificaciones y limtaciones del sistema.
    3. Diseño: Se planean la arquitectura y tecnologías a utilizar.
    4. Desarrollo: Se crea el sistema, según lo establecido en los pasos anteriores.
    5. Pruebas: Se comprueba de que el sistema funcione correctamente, cumpliendo con lo qué se buscaba y no teniendo fallos o errores que impidan su buen uso.
    6. Implementación: El sistema comienza a ser operativo y usado por el público.
    7. Mantenimiento: Durante su vida útil, se actualiza el sistema cuando las circunstancias lo requiere y se corrigen errores que puedan surgir a futuro.
    8. Retiro: El sistema es retirado, sus datos son archivados o trasladados a otro sistema, si existe tal sucesor.
